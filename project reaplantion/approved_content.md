# What is `exports/approved_content.json`?

`exports/approved_content.json` is the **publication-ready data snapshot** generated by the export pipeline.

It is not the full database dump.  
It is a curated export intended for:

- website rendering of approved content,
- content handoff to other systems,
- print/book pipeline preprocessing.

## Why this file exists

The project rule is: **only approved content can be published**.

So this file acts as a safe output layer between:

- internal working DB (contains pending/rejected items), and
- public/production content surfaces.

## How it is generated

It is created by:

- `innahu_allah/exporters/approved_export.py`
- command: `uv run python -m innahu_allah.cli export-approved`

This command writes:

- `exports/approved_content.json`
- `exports/approved_book.md`

## What it contains (current structure)

Top-level keys:

- `names`
- `verses`
- `tafsir_entries`
- `hadiths`
- `scholarly_commentary`
- `father_narrative`

## Filtering logic used

- `tafsir_entries`: only `verification_status = approved`
- `hadiths`: only `verification_status = approved`
- `scholarly_commentary`: only `verification_status = approved`
- `father_narrative`: only `status = final`
- `verses`: verses linked to names (for current slice)
- `names`: approved names or names participating in linked verse context

This matches the governance rule (`approved_only`) while still allowing complete page rendering context.

## Why there is `fatihah_seed` and `seed_demo` inside

Current data is from the implemented Al-Fatihah seed pipeline (`seed_fatihah.py`) used to validate end-to-end flow before live API ingestion.

- `source_ref: fatihah_seed`
- `data_source: seed_demo`

These fields are intentional provenance markers, so everyone can see this is seeded/demo ingestion, not final live-source harvest.

## How to read one record

Each object includes content + provenance metadata:

- content fields (e.g., `ayah_text`, `tafsir_text`, `arabic_text`)
- review fields (`verification_status`, `reviewer_notes`)
- provenance fields (`source_ref`, `source_url`, `ingestion_timestamp`, `parser_version`, `source_hash`)

This supports traceability and auditability required by the plan.

## Final meaning of this file

`approved_content.json` is the **single trusted JSON output** for approved publication content at the current project stage (Al-Fatihah MVP slice).
